Methane Detection


Overview:

The challenge at hand involved identifying and localizing methane leaks in the atmosphere using a dataset comprising 64x64 grayscale satellite images from diverse locations. The primary goal was to develop a robust deep learning model capable of accurately discerning whether a specific location exhibits a methane plume or not. This was a team effort along with other contributors including Congjie AN, Vanshika Jain, Oluwafisayomi BADARU, Tun√ß EREN and Corentin DELLOYE.

Approach:

  • Pre-processing & Data Loading: Initial steps included image loading, normalization, label extraction, and partitioning the dataset into training and testing sets.

  • Data Augmentation: Followed by that, we augmented the training set using techniques like image rotation, flipping, and random noise addition, and employed PyTorch Datasets & DataLoaders for streamlined processing.

  • CNN Architecture: Next, we designed a Convolutional Neural Network (CNN) architecture, comprising a max pooling layer for dimension reduction and two fully connected layers for classification, defining the forward method to guide the image through these layers.

  • Training & Validation: After designing the model structure, we conducted training and validation using an 80-20 split of the training set to fine-tune the model’s performance.

  • Model Evaluation: Finally, we assessed the final model’s performance on the testing set, achieving notable metrics in accuracy (77%), F1 score(0.79), and AUC(87%).

  • Developing Backend UI with Flask: Last but not the least, we planned and developed a user-friendly interface (UI) with the built model at the backend using the Flask framework. This interface fcilitated model access and interaction for users handling methane emission detection tasks.

Outcomes:

We identified methane plumes in 44.4% of the 108 total images, primarily concentrated in Asian regions, notably India, Bangladesh, and Pakistan.

Impact and Significance:

I believe that the built framework can be efficently utilised for below use cases:

  • Industrial Monitoring: Enabling oil and gas companies to monitor infrastructure for real-time leak detection and rapid response, minimizing environmental impact.

  • Regulatory Compliance: Assisting environmental agencies in monitoring methane-producing sites and ensuring adherence to environmental regulations.

  • Environmental Research: Aiding climate scientists in understanding methane emission sources for improved climate models and forecasts.

  • Leak Prevention: Empowering companies with insights to enhance infrastructure and prevent potential future leaks.

  • Carbon Credits Market: Validating methane reduction efforts for companies seeking carbon offsets, bolstering the integrity of the carbon credits market.

  • Public Awareness: Utilizing an interactive map displaying detected methane leaks to educate the public about the scale and impact of methane emissions.

Conclusion:

This project provides a comprehensive approach that demonstrates the effectiveness of AI in methane detection and an implementation of a user interface using Flask to streamline backend interactions with the developed model.
